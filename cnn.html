Interactive CNN classifier for 3×3 binary images

The task is binary classification. A pixel value of 1 means black. The target is 1 when the 3×3 image contains a complete line of three black pixels: by default, a full row or a full column. The CNN is intentionally small: valid 3×3 convolution filters → ReLU → sigmoid output. Because the image is only 3×3, each learned filter acts as a directly visible template detector.

1. Full dataset

All possible 3×3 binary images exist in the dataset. Click any thumbnail to inspect it.

2. Target rule

The label is 1 when at least one of these templates is fully black inside the image.

3. Selected 3×3 image

Click cells to create a custom image. Black is value 1, white is value 0.

4. Learned CNN filters

Each 3×3 filter is a learned template. Click a filter to inspect its computation.

5. Selected filter computation

The selected image is multiplied by the selected filter, summed with a bias, then passed through ReLU.

6. Filter responses

ReLU activations are the detected features. The output layer combines them into one probability.

7. CNN forward path

Input pixels feed every filter. ReLU keeps positive filter responses. The sigmoid maps the final logit to class 1 probability.

8. Backpropagation view

Signed gradients show which pixels and selected-filter weights would most change the current loss.

9. Training curve

Loss should decrease and accuracy should approach 100% on the exhaustive 512-image dataset.

Architecture: 3×3 input → multiple trainable 3×3 convolution filters → ReLU → weighted sum → sigmoid. With a 3×3 input, a valid 3×3 convolution produces one scalar response per filter. This makes the CNN behave like a bank of learned pattern detectors. During backpropagation, the sigmoid gives an error term p − y, the output weights send that error back into the active ReLU filters, and inactive ReLUs block their gradient.
